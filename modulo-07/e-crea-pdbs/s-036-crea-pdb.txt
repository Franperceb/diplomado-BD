**CONCLUSIONES
Este ejercicio ilustra el proceso de creación y uso de Pluggable Databases (PDBs) tipo refreshable. 
Inicia con la configuración de una conexión entre dos Container Databases (CDBs) y la creación de una 
PDB refreshable en una CDB utilizando un Database Link. 
Luego, se realiza una consulta para verificar el último SCN (System Change Number) de refresh, 
lo que permite monitorear el estado de la PDB refreshable. A continuación, 
se crean datos de prueba en la PDB origen y se abren para lectura en la PDB destino. 
Se observa el comportamiento de los datos creados en la PDB origen al realizar un refresh desde la CDB root. 
Esto muestra cómo los cambios en la PDB origen se reflejan en la PDB destino después de un refresh. 
Finalmente, se realizan las operaciones de limpieza adecuadas, como el cierre y eliminación de la PDB, 
la eliminación del Database Link y la limpieza de los usuarios y tablespaces creados durante el ejercicio.
Este ejercicio resalta la utilidad de las PDBs refreshable para mantener datos actualizados entre diferentes CDBs 
de manera eficiente y automatizada, lo que facilita la gestión y el mantenimiento de bases de datos distribuidas.
